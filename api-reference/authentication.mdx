## Overview

Deni AI uses [better-auth](https://www.better-auth.com/) for authentication, providing a comprehensive set of endpoints for user authentication, session management, and account security.

All authentication endpoints are available at:

```
POST/GET /api/auth/[...all]
```

## Authentication Methods

Deni AI supports multiple authentication methods:

### Email & Password

Traditional email and password authentication with:
- Email verification (when RESEND_API_KEY is configured)
- Password reset via email
- Have I Been Pwned integration for password security checking

### OAuth Social Providers

**Google OAuth**
- Provider: `google`
- Requires: `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET`

**GitHub OAuth**
- Provider: `github`
- Requires: `GITHUB_CLIENT_ID` and `GITHUB_CLIENT_SECRET`

### Magic Link

Passwordless authentication via email (when `RESEND_API_KEY` is configured):
- One-click sign-in links sent to email
- No password required
- Secure time-limited tokens

### Passkey (WebAuthn)

Modern passwordless authentication using:
- Biometric authentication (fingerprint, Face ID)
- Hardware security keys
- Platform authenticators

### Anonymous Sessions

Guest access without registration:
- Instant access for trying the platform
- Limited features (no premium models)
- Can be converted to permanent accounts

### Two-Factor Authentication (2FA)

Additional security layer:
- TOTP (Time-based One-Time Password)
- Backup codes
- Per-user optional enablement

## API Keys for Programmatic Access

For API access, generate API keys in your account:

1. Keys must start with `deni_` prefix
2. Include in requests: `Authorization: Bearer deni_your_key`
3. Keys can have expiration dates
4. Track last usage timestamp
5. Can be revoked at any time

### API Key Authentication Flow

```javascript
// Making authenticated requests
fetch('https://your-domain.com/api/chat', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer deni_abc123...',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ /* request data */ })
});
```

### API Key Validation

When an API key is provided:
1. System validates the `deni_` prefix
2. Hashes the key and looks up in database
3. Checks if key exists and hasn't expired
4. Extracts user ID for request authorization
5. Updates `lastUsedAt` timestamp asynchronously

### Session Authentication

For browser-based applications, sessions are managed via HTTP-only cookies:

```javascript
// better-auth client automatically handles session cookies
import { authClient } from '@/lib/auth-client';

// Sign in
await authClient.signIn.email({
  email: 'user@example.com',
  password: 'password123'
});

// Session cookie is automatically set
// Subsequent requests include session automatically
```

The `/api/auth/[...all]` catch-all route handles all better-auth endpoints. Common endpoints include:

### Sign In/Sign Up

```
POST /api/auth/sign-in/email

POST /api/auth/sign-in/social
```

```

```

```

```

## Rate Limiting

Authentication endpoints have strict rate limits:

### Sign-In Endpoints
- **Window**: 10 seconds
- **Max Requests**: 3 requests
- **Applies to**: `/sign-in/*` routes

### Two-Factor Endpoints
- **Window**: 10 seconds
- **Max Requests**: 3 requests
- **Applies to**: `/two-factor/*` routes

### General Auth Endpoints
- **Window**: 60 seconds
- **Max Requests**: 100 requests
- **Applies to**: All other auth endpoints

## Client Integration

Use the better-auth client for browser applications:

```typescript
// src/lib/auth-client.ts
import { createAuthClient } from "better-auth/react";

export const authClient = createAuthClient({
  baseURL: "http://localhost:3000"
});
```

### Example Usage

```typescript
import { authClient } from '@/lib/auth-client';

// Email sign in
await authClient.signIn.email({
  email: 'user@example.com',
  password: 'password'
});

// Google OAuth
await authClient.signIn.social({
  provider: 'google'
});

// Get current session
const session = await authClient.getSession();

// Sign out
await authClient.signOut();
```

## Server-Side Session Access

In API routes or server components:

```typescript
import { auth } from '@/lib/auth';
import { headers } from 'next/headers';

const headersList = await headers();
const session = await auth.api.getSession({ headers: headersList });

const userId = session?.session?.userId;
const isAnonymous = Boolean(session?.user?.isAnonymous);
```

## Best Practices

1. **Use API Keys for Programmatic Access**: Better suited for server-to-server and API clients
2. **Use Sessions for Browser Apps**: Leverage HTTP-only cookies for web applications
3. **Enable 2FA**: Encourage users to enable two-factor authentication
4. **Rotate API Keys**: Regularly rotate API keys and set expiration dates
5. **Monitor Rate Limits**: Implement exponential backoff for rate-limited requests
6. **Validate on Server**: Always validate authentication server-side, never trust client data
